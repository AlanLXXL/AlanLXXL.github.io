# General Info

## 生活部分

* 从伦敦希思罗往返诺丁汉个人看法：  
  * 人数多建议包车，火车或者大巴没有包车方便，其次刚到英国大概率还办没购买学生的Railcards, 火车票价格较贵  
  * 包车公司比较多，我们之前用的是长海车服，大家也可以自行小红书等平台搜索，英国人工费贵，如果遇到特别便宜的大概率不靠谱

* 猪不理校区是有食堂的 （Atruim)  
  * 中饭固定窗口是薯条，鸡肉，卷饼  
  * 中饭另外一个窗口每天食物轮流  
  * 中饭平均价格4磅左右  
  * 晚饭价格固定6磅  
    * 固定一份主食，其余所有食物自助，吃多少拿多少，有甜品，饮料

* 诺丁汉体育氛围浓厚，Park校区和猪不理校区都有体育馆，Park校区体育馆覆盖基本所有运动场地，包括游泳池等，部分场地或者活动类似宁诺，需要提前预约。学校体育馆需要付费成为会员才能享受大部分权益，不同于宁诺，有早鸟价（一般在开学前几周）

* 诺丁汉好吃的市中心相对比较多，距离猪不理校区20分钟车程，可以自行探索

* 诺丁汉的超市市中心的Waitrose 和 M&S 属于价格最贵的 （品质类型），其次是 Tesco 和 Sainsbury, 最后是 Lidl 和 Aldi （实惠类型），大部分都可以送货上门（需满足一定金额，40磅左右)

* 诺丁汉的中超比如得意行等里面东西也很全，大部分调料都能买到

  * 雅思或者GRE要不要考取决于申请项目: 不同到项目要求不一样，也不用过分焦虑。2+2还是可以帮助我们waive掉很大一部分的语言的。总体上说往年2+2美国可以waive掉大部分语言，英国比如IC是waive不掉的，

    * 个人备考是听力刷一部分真题，阅读 (刘洪波的阅读真经总纲+考前刷一部分真题), 写作 B站Simon 写作， 口语以真题为主  
  * GRE备考周期会比较长（330以上我觉得除非天赋异禀，大部分人还是要准备两个月左右 )，且和英语基础关系不多，GRE单词关一定要过，建议背GRE3000词，起码滚五轮及以上（我就是当时一个月备考来不及了，只背了核心1400词，考试时候碰到medium或者hard题很多单词还是不认识或者意思掌握错）  
  * GRE的备考痛苦程度比雅思大很多，不确定要不要去考的可以先去看下它的Verbal 题目，不要盲目报考

# Course Introduction

## Software Engineering Group Project (全年项目，共20学分）

这门课是全年的项目类课程，从项目前期的需求明确到中间开发到最后的汇报持续两个学期。

* 个人认为比较有用的几个点：  
  * 开学时会知道小组分组，一组一般8个人，英诺算法一般会让组里有两个或者以上中国人，但不绝对。有三个职位（Team leader, Team admin, Git Leader)，组里自行分配  
    * 三个职位宽泛概念：  
      * Team leader 负责领导团队  
      * Team admin 有选择项目，上交全组报告等Moodle权限  
      * Git leader 负责管理代码  
  * Team Admin 需要一个比较靠谱的人，需要负责帮助全组抢项目，同时需要负责按时上交全组项目代码  
  * Team Leader 需要和全组有效沟通，推进项目，不然可能出现项目就两三个人在做，其他小组成员摆烂或者跟不上进度  
  * Git leader 相对上面两个没那么重要，主要负责管理repository

* 个人看法 （仅基于我的个人认知和了解到的经历）  
  * 英诺GRP项目有和企业合作，也老师的项目，项目之间侧重点和所要求的技术差距很大，能选到心仪项目是好事，没选到也没关系，项目分配有比较大的不确定性。  
  * 英诺GRP最后给分相关负责老师会进行调控，如果项目进展不顺利也不要灰心，尽力去做就行。  
  * GRP项目需要积极和国外组员沟通，不用特别担心英语，对于native speaker来说只要你关键词，关键语句表达出来了，基本都能知道你表达的意思。

## Operating Systems & Concurrency (20学分）

这门课是学习操作系统，全课程分为Processes，Concurrency, Memory Management, File Systems 四部分展开，50%CW， 50%考试

* 个人认为比较有用的几个点：  
  * OSC的50%大CW一般截止时间在圣诞假期之后，需要合理安排时间，个人建议别只预留两三天。  
  * 大作业的难度相比大三其他的作业会比较大，如果对于操作系统不熟悉，需要多预留一些时间完成，作业用C语言，并且完成过程中可能会碰到死锁等问题，debug需要较多时间。  
  * 期末考一般是四个topic选三个回答，期末复习时可以有侧重。

* 个人看法 （仅基于我的个人认知和了解到的经历）  
  * 这门课可以说是大三的一门比较硬的课，偏向底层意味着课程难度和抽象要求都比较高，如果没相关基础建议CW早点开始，一般圣诞假期就很难当面问老师问题了，邮件回复频率也会降低。  
  * 英诺CW一般都会在圣诞假期，圣诞假期后有一个复习周，之后就期末考试，这门课的复习量相比大三上其他课更多，需要合理安排时间

## Developing Maintainable Software （20学分）

这门课会学习软件工程里会用到的相关开发原则，整门课围绕如果开发可维护项目展开，使用java

个人看法 （仅基于我的个人认知和了解到的经历）

* 这门课从2023年开始换了新的老师，改了课程，听说2025年课程又完全从头设计了，所以不清楚2025届课程设计会怎么样。  
* 这门课不同于其他课程，评判标准涉及到对于开发程序的各方面进行打分，导致评分原则会相对比较主观  
* 这门课这两届开始上80分难度非常非常大，并且80分以上评分细则主观性很强，我们这届22和40过来的这门课CW上80的我了解到的没有。我个人建议是需要调整心态，如果时间兼顾不过来的话，别投入太多太多时间在最后的80到100的评分里面，最后的效果还会变差。

## Introduction to Formal Reasoning （10学分）

简单逻辑学和计算机程序正确性论证，使用Lean

个人看法 （仅基于我的个人认知和了解到的经历）

* 这门课的质量非常非常高，完善的lecture notes, 每周同步的小练习，小班化的tutorial，以及英诺这里的Professor Thorsten 亲自授课，上课幽默，上课采取敲代码和手写板书的方式上课，上课节奏不是很快便于跟着敲代码和理解。  
* 这门课的tutorial建议大家每周都要去听，是小班化教学，一个老师对10多个学生。我的tutorial 老师是Mark Williams, 是Thorsten下面的博士生，牛津数学系本硕，讲课水平很高，同时对于学生非常友好，所有问题都会回答。  
* 题外话：在这门课里你会慢慢感受到英诺计算机的强势领域下面的课程是怎么样设计和教学的。如果你对functional programming和程序证明有兴趣，那么英诺functinal programming 组可以去了解下。英诺这个组在英国属于最顶尖的，他们也很欢迎本科生去参与他们的日常seminar讨论。

# Summary

大三在英国的一年相比国内肯定是有更多的挑战的。英国一个陌生的环境和冬令时的影响都会让人的心态产生波动。

你们会看到很多在英国留学的建议，我个人觉得每天适量的锻炼，一定的社交，好好吃饭，好好睡觉其实比什么都要重要。

这两年宁诺越来越多的人开始犹豫是否选择22，或者是否选择到英诺来交换。我个人觉得既然我们已经选择了大三来英诺学习，就不要在大三一直去和之前宁诺的经历做比较。留学的途中很多东西是没有办法进行比较的，留学途中个人的成长，个人的经历都是独一无二的，从自己烧的每一道菜，到和同伴一起规划的游玩，许多东西只有体验了以后才能慢慢明白。

在英诺如果有任何的疑问，都可以去向身边同学或者学长学姐求助。在宁诺和英诺的学习从来都不是一个人的单打独斗，我们宁诺计算机系从最开始的申请分享会和后面一直延续的飞跃手册，都离不开每一届同学间的相互鼓励和学长学姐的热心帮助。

我到现在还记得我大一时郭方骋学长分享时的一句话：知无不言，言无不尽。  
希望大家也能够在后续的学习旅程中去更多的帮助到周围的同学和学弟学妹。

最后祝愿每个人在大三的旅程中都能越来越好❤️
